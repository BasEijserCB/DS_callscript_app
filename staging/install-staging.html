DS Logboek — Staging

Test build

Sleep de oranje knop hieronder naar je bookmarkbalk om de functionele staging build te installeren. Hij staat los van je productie bookmarklet — beide blijven werken.

↻ DS Logboek (staging) ← sleep naar bookmarkbalk
⚠️ Voor je sleept: controleer in loader-staging-bookmarklet.js dat SCRIPT_URL naar de raw GitHub URL van staging/ds-logboek-staging.js wijst. Plak daarna de geminifyde loader in het href-attribuut van de knop hierboven (zie stap 2).

Installatie — 4 stappen

1. Push staging/ds-logboek-staging.js naar je GitHub repo. Zorg dat de raw URL publiek bereikbaar is.
2. Minify loader-staging-bookmarklet.js (bv. via esbuild --minify of een online minifier). Zet er javascript: voor en plak het resultaat als href in de oranje knop hierboven.
3. Sleep de oranje knop naar je bookmarkbalk, naast je bestaande blauwe prod-loader.
4. Open een DireXtion order, klik de staging bookmarklet. De functionele staging-widget verschijnt los van productie.

A/B test workflow

  • Productie: blauwe knop → huidige ds-logboek.js
  • Staging: oranje knop → functionele ds-logboek-staging.js
  • Aparte localStorage caches (ds_app_prod_cache vs. ds_app_staging_cache_v3)
  • Aparte DOM-wrapper (#ds-combi-staging-wrapper) — geen conflict
  • Beide gebruiken hetzelfde paste-bookmarklet (clipboard schema is identiek)
