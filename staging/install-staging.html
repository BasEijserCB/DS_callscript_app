DS Logboek — Staging

Test build

Sleep de oranje knop hieronder naar je bookmarkbalk om de nieuwe design preview te installeren. Hij staat los van je productie bookmarklet — beide blijven werken.

↻ DS Logboek (staging) ← sleep naar bookmarkbalk

Installatie — 2 stappen

1. Sleep de oranje knop naar je bookmarkbalk, naast je bestaande blauwe prod-loader.
2. Open een DireXtion order, klik de staging bookmarklet → het nieuwe side panel verschijnt linksonder.

A/B test workflow

  • Productie: blauwe knop → huidige ds-logboek.js
  • Staging: oranje knop → nieuwe ds-logboek-staging.js
  • Aparte localStorage caches (ds_app_prod_cache vs. ds_app_staging_cache)
  • Aparte DOM-roots (#ds-logboek-staging-root) — geen conflict
  • Beide gebruiken hetzelfde paste-bookmarklet (clipboard schema is identiek)

Wat de staging build doet (v0.3.0)

  • Volledige DOM-scrape van DireXtion orderdata (beide varianten)
  • Volledige gespreksflow — CBB, CBF, Teamleider, Andere bellers, Algemeen gesprek
  • Echte clipboard payload (identiek schema als paste-bookmarklet)
  • Loggen naar Google Sheets via GAS
  • Sessie park / herstel (localStorage)
  • Nieuw side-panel design — linksonder, React + Babel
